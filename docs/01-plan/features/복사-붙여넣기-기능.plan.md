# [Plan] 복사-붙여넣기 기능

> **Feature**: Copy-Paste Support for Bulk Ticket Creation
> **PDCA Phase**: Plan
> **Created**: 2026-02-25
> **Status**: In Planning

---

## 1. 개요 (Overview)

### 1.1 배경
현재 Jira Bulk Creator의 Create 탭에서 티켓을 생성하려면 **모든 필드를 하나씩 수동으로 입력**해야 합니다. 그러나 PM들은 이미 Excel, Google Sheets, Confluence 표에 티켓 정보를 정리해두는 경우가 많습니다.

### 1.2 문제점
- Excel/Google Sheet에서 데이터를 복사한 후, **일일이 필드를 클릭해서 붙여넣기** 해야 함
- 10개 이상의 티켓을 입력할 때 반복 작업으로 인한 생산성 저하
- Confluence 표에서 직접 가져올 수 없어 수작업 전환 필요

### 1.3 솔루션
Excel/Google Sheets/Confluence 표에서 **Ctrl+V 한 번**으로 여러 행을 일괄 입력할 수 있는 복사-붙여넣기 기능 구현.

---

## 2. 목표 (Goals)

### 2.1 Primary Goal
- **생산성 향상**: 10개 티켓 입력 시간을 10분 → 1분으로 단축 (90% 감소)

### 2.2 Secondary Goals
- Excel/Google Sheet/Confluence 표 형식 완벽 지원
- 기존 행에 덮어쓰기 또는 새 행 추가 선택 가능
- 데이터 매핑 자동 감지 (열 순서 매칭)

### 2.3 Non-Goals (이번 구현에서 제외)
- CSV 파일 드래그 앤 드롭 (별도 기능으로 이미 존재)
- 이미지/첨부파일 복사-붙여넣기
- 열 헤더 자동 감지 (수동 매핑만 지원)

## 3. 범위 (Scope)

### 3.1 In-Scope

#### 3.1.1 지원 데이터 소스
| 소스 | 형식 | 우선순위 |
|------|------|----------|
| **Excel** | Tab 구분, 개행 구분 | 높음 (필수) |
| **Google Sheets** | Tab 구분, 개행 구분 | 높음 (필수) |
| **Confluence 표** | Tab 구분, 개행 구분 | 중간 |
| **Notion 표** | Tab 구분, 개행 구분 | 낮음 (Bonus) |

#### 3.1.2 붙여넣기 모드
1. **덮어쓰기 모드**: 선택된 셀부터 시작하여 기존 데이터 덮어쓰기
2. **삽입 모드**: 선택된 행 위치에 새 행 삽입 후 데이터 채우기

#### 3.1.3 데이터 매핑
- **자동 매핑**: 클립보드 데이터의 열 순서를 테이블 열 순서에 맞춰 자동 매핑
- **예시**:
  ```
  복사된 데이터: "Epic\t캐릭터 스킬\t스킬 시스템 구현\t홍길동\tSprint 1"
  → 유형(Epic), 제목(캐릭터 스킬), 설명(스킬 시스템 구현), 담당자(홍길동), Sprint(Sprint 1)
  ```

#### 3.1.4 데이터 검증
- 필수 필드 누락 시 경고 표시 (제목 없음)
- 유효하지 않은 유형 값 (Epic/Task 외) 자동 수정 또는 경고
- 담당자/Sprint 값이 목록에 없으면 경고 (생성은 허용)

### 3.2 Out-of-Scope
- 다중 선택 영역 붙여넣기 (단일 시작 셀만 지원)
- 수식(formula) 붙여넣기
- 셀 병합 처리
- 서식(색상, 글꼴) 복사

## 4. 요구사항 (Requirements)

### 4.1 Functional Requirements

#### FR-1: 클립보드 데이터 파싱
**우선순위**: 높음
**설명**: `clipboard.getData('text/plain')`으로 텍스트 데이터를 가져와 Tab(\t)과 개행(\n)으로 파싱

**입력 예시**:
```
Epic\t캐릭터 스킬 시스템\t스킬 기획 및 구현\t홍길동\tSprint 1\t2026-03-01\t2026-03-31\t
Task\t[기획] 스킬 기획서\t스킬 종류 정의\t김기획\tSprint 1\t\t\tAEGIS-100
```

**출력 예시**:
```javascript
[
  {
    type: 'Epic',
    summary: '캐릭터 스킬 시스템',
    description: '스킬 기획 및 구현',
    assignee: '홍길동',
    sprint: 'Sprint 1',
    startDate: '2026-03-01',
    dueDate: '2026-03-31',
    parentKey: ''
  },
  {
    type: 'Task',
    summary: '[기획] 스킬 기획서',
    description: '스킬 종류 정의',
    assignee: '김기획',
    sprint: 'Sprint 1',
    startDate: '',
    dueDate: '',
    parentKey: 'AEGIS-100'
  }
]
```

#### FR-2: 테이블 행 업데이트
**우선순위**: 높음
**설명**: 파싱된 데이터를 선택된 셀부터 시작하여 테이블에 채움

**동작**:
1. 사용자가 특정 셀 클릭 (예: 2행 3열)
2. Ctrl+V 입력
3. `handlePaste(event, startRowIndex=2, startColIndex=3)` 호출
4. 2행 3열부터 데이터 채우기 시작

**행 추가 로직**:
- 복사된 데이터 행 수가 기존 행 수보다 많으면 자동으로 새 행 추가

#### FR-3: 데이터 타입 변환
**우선순위**: 중간
**설명**: 문자열 데이터를 각 필드 타입에 맞게 변환

| 필드 | 타입 | 변환 로직 |
|------|------|-----------|
| 유형 | Dropdown | "Epic", "Task" 외 값 → 기본값 "Task" |
| 담당자 | Dropdown | displayName → accountId 매핑 (캐시 조회) |
| Sprint | Dropdown | Sprint 이름 → Sprint ID 매핑 |
| 날짜 | Date | YYYY-MM-DD 형식 검증, 잘못된 형식 시 빈 값 |

#### FR-4: 사용자 피드백
**우선순위**: 중간
**설명**: 붙여넣기 결과 사용자에게 피드백

**Toast 알림**:
- 성공: "✅ 5행 붙여넣기 완료"
- 경고: "⚠️ 3행 붙여넣기 완료 (2개 필드 누락)"
- 실패: "❌ 붙여넣기 실패: 데이터 형식 오류"

### 4.2 Non-Functional Requirements

#### NFR-1: 성능
- 100행 붙여넣기 시 1초 이내 처리
- 1000행 이상 시 경고 메시지 ("대량 데이터는 CSV 업로드 권장")

#### NFR-2: 호환성
- Chrome, Edge, Firefox 최신 버전 지원
- Clipboard API 사용 (`navigator.clipboard` 또는 `ClipboardEvent`)

#### NFR-3: 접근성
- 키보드만으로 전체 작업 가능 (Ctrl+V)
- 스크린 리더 지원 (aria-label)

## 5. 제약사항 (Constraints)

### 5.1 기술적 제약사항
- **브라우저 제약**: 일부 구형 브라우저는 Clipboard API 미지원 → Fallback으로 `document.execCommand('paste')` 사용 불가능 (보안 제약)
- **클립보드 형식**: 텍스트만 지원, 이미지/파일 제외
- **동시 편집**: 한 번에 하나의 붙여넣기 작업만 가능 (동시성 제어 없음)

### 5.2 설계 제약사항
- **기존 구조 유지**: `SpreadsheetTable.tsx` 컴포넌트 구조 변경 최소화
- **Zustand Store 사용**: `useTicketStore`의 `updateRow()`, `addRow()` 액션 재활용

### 5.3 시간 제약사항
- **개발 기간**: 2일 (계획 0.5일, 구현 1일, 테스트 0.5일)

## 7. 설계 방향 (Design Direction)

### 7.1 아키텍처 개요

```
┌─────────────────────────────────────────────────┐
│  User (Excel/Google Sheets/Confluence)          │
└───────────────┬─────────────────────────────────┘
                │ Ctrl+C (Copy)
                ▼
┌─────────────────────────────────────────────────┐
│  Clipboard (Browser)                            │
└───────────────┬─────────────────────────────────┘
                │ Ctrl+V (Paste Event)
                ▼
┌─────────────────────────────────────────────────┐
│  SpreadsheetTable.tsx                           │
│  - handlePaste(event, rowIndex, colIndex)       │
└───────────────┬─────────────────────────────────┘
                │
                ▼
┌─────────────────────────────────────────────────┐
│  lib/clipboard.ts (NEW)                         │
│  - parseClipboardData(text): ParsedRow[]        │
│  - validateRow(row): ValidationResult           │
│  - mapFieldValue(field, value): any             │
└───────────────┬─────────────────────────────────┘
                │
                ▼
┌─────────────────────────────────────────────────┐
│  useTicketStore (Zustand)                       │
│  - updateRow(id, data)                          │
│  - addRow(data)                                 │
│  - bulkUpdateRows(rows) (NEW)                   │
└─────────────────────────────────────────────────┘
```

### 7.2 주요 컴포넌트

#### 7.2.1 SpreadsheetTable.tsx (수정)
```typescript
function SpreadsheetTable() {
  const { rows, bulkUpdateRows } = useTicketStore();

  const handlePaste = useCallback((event: ClipboardEvent, startRow: number, startCol: number) => {
    event.preventDefault();
    const text = event.clipboardData?.getData('text/plain');
    if (!text) return;

    const parsedData = parseClipboardData(text);
    const updatedRows = mapDataToRows(parsedData, startRow, startCol, rows);
    bulkUpdateRows(updatedRows);

    toast.success(`✅ ${parsedData.length}행 붙여넣기 완료`);
  }, [rows, bulkUpdateRows]);

  // ...
}
```

#### 7.2.2 lib/clipboard.ts (신규)
```typescript
export interface ParsedRow {
  type: string;
  summary: string;
  description: string;
  assignee: string;
  sprint: string;
  startDate: string;
  dueDate: string;
  parentKey: string;
}

export function parseClipboardData(text: string): ParsedRow[] {
  const lines = text.split('\n').filter(line => line.trim());
  return lines.map(line => {
    const cells = line.split('\t');
    return {
      type: cells[0] || 'Task',
      summary: cells[1] || '',
      description: cells[2] || '',
      assignee: cells[3] || '',
      sprint: cells[4] || '',
      startDate: cells[5] || '',
      dueDate: cells[6] || '',
      parentKey: cells[7] || ''
    };
  });
}

export function validateRow(row: ParsedRow): ValidationResult {
  const errors: string[] = [];
  const warnings: string[] = [];

  if (!row.summary) errors.push('제목 필수');
  if (row.type !== 'Epic' && row.type !== 'Task') warnings.push('유효하지 않은 유형');

  return { isValid: errors.length === 0, errors, warnings };
}
```

### 7.3 데이터 흐름

```
1. User pastes data (Ctrl+V)
   ↓
2. onPaste event captured on focused cell
   ↓
3. parseClipboardData(text)
   - Split by '\n' and '\t'
   - Create ParsedRow[]
   ↓
4. validateRow() for each row
   - Collect errors/warnings
   ↓
5. mapDataToRows()
   - Convert displayName → accountId
   - Convert Sprint name → Sprint ID
   - Convert date strings → Date objects
   ↓
6. bulkUpdateRows(updatedRows)
   - Zustand updates state
   - IndexedDB auto-saves via persist middleware
   ↓
7. UI re-renders with new data
   ↓
8. Show toast notification
```
