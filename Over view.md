# Jira Bulk Creator - 기획서 (바이브 코딩용)



## 1. 프로젝트 개요



| 항목 | 내용 |

|------|------|

| **명칭** | Jira Bulk Creator |

| **목적** | PM을 위한 Jira 티켓 일괄 생성 및 이력 관리 |

| **플랫폼** | 웹 애플리케이션 (브라우저 기반) |



### 1.1 해결하려는 문제



| 현재 문제 | 해결 방안 |

|-----------|-----------|

| 티켓을 하나씩 수작업 생성 | 스프레드시트 형태 UI로 일괄 입력/생성 |

| Excel → Confluence → MCP 다단계 작업 | 직접 복붙으로 단일 단계 처리 |

| 업무 유형별 폼이 제각각 | Epic 하위 일감 템플릿으로 표준화 |

| 생성 이력 추적 불가 | 기록 탭에서 이력 관리 |

## 4. 생성 탭



### 4.1 레이아웃



```

┌─────────────────────────────────────────────────────────────────────────────────────┐

│ 추가된 티켓 (3/100)                        [도움말] [내보내기] [업로드(.csv)]       │

├─────────────────────────────────────────────────────────────────────────────────────┤

│    │유형   │제목        │설명     │담당자  │Sprint  │시작일  │기한    │상위업무│     │

├────┼───────┼────────────┼─────────┼────────┼────────┼────────┼────────┼────────┼─────┤

│ ☑  │Task ▼│           │         │      ▼ │      ▼ │        │        │        │[복사]│

├────┼───────┼────────────┼─────────┼────────┼────────┼────────┼────────┼────────┼─────┤

│ ☑  │Epic ▼│           │         │      ▼ │   -    │        │        │   -    │[+하위]│

├────┼───────┼────────────┼─────────┼────────┼────────┼────────┼────────┼────────┼─────┤

│ ☑  │Task ▼│           │         │      ▼ │      ▼ │        │        │        │[복사]│

├────┴───────┴────────────┴─────────┴────────┴────────┴────────┴────────┴────────┴─────┤

│                                    [ + 행 추가 ]                                     │

├─────────────────────────────────────────────────────────────────────────────────────┤

│ [🗑 선택 삭제]                                          [미리보기]  [▶ 생성 시작]   │

└─────────────────────────────────────────────────────────────────────────────────────┘

```



### 4.2 테이블 필드 스펙



| 필드명 | 타입 | 필수 | 기본값 | 설명 |

|--------|------|------|--------|------|

| 선택 | Checkbox | - | true | 행 선택용 |

| 유형 | Dropdown | O | Task | Epic, Task |

| 제목 | Text Input | O | - | 최대 255자 |

| 설명 | Textarea | - | - | 자동 높이 조절 |

| 담당자 | Dropdown | - | - | Jira 사용자 목록 |

| Sprint | Dropdown | - | - | 활성 Sprint 목록 |

| 시작일 | Date Picker | - | - | YYYY-MM-DD |

| 기한 | Date Picker | - | - | YYYY-MM-DD |

| 상위업무 | Text Input | - | - | Epic Key (예: AEGIS-100) |

| 액션 | Button | - | - | 유형에 따라 다름 |



### 4.3 유형별 액션 버튼



| 유형 | 버튼 | 동작 |

|------|------|------|

| **Task** | [복사] | 바로 위 행의 모든 내용을 현재 행에 복사 |

| **Epic** | [+하위] | 하위 일감 추가 모달 열기 |



### 4.4 입력 기능



| 기능 | 동작 |

|------|------|

| **직접 입력** | 각 셀 클릭하여 입력, 내용에 따라 셀 높이 자동 조절 |

| **복사-붙여넣기** | Excel/Google Sheet/Notion/Confluence 표에서 복사 → 셀 선택 → Ctrl+V |

| **행 추가** | [+ 행 추가] 버튼 또는 마지막 행에서 Tab/Enter |

| **행 삭제** | 체크박스 선택 후 [선택 삭제] |

| **업로드** | CSV 파일 업로드 시 테이블에 자동 채움 |

| **내보내기** | 현재 테이블 내용을 CSV로 다운로드 |

## 8. 생성 로직



### 8.1 생성 순서



```

1. 입력된 행을 순회

2. Epic 먼저 모두 생성 → 생성된 Jira Key 저장

3. Task 생성 시:

   - parentKey가 있으면 그대로 사용 (예: AEGIS-100)

   - parentRowId가 있으면 해당 Epic의 생성된 Key로 대체

4. 각 생성 결과를 기록에 저장

```



### 8.2 Jira API 호출 스펙



**Epic 생성**

```

POST /rest/api/3/issue

{

  "fields": {

    "project": { "key": "AEGIS" },

    "issuetype": { "name": "Epic" },

    "summary": "제목",

    "description": { "type": "doc", "version": 1, "content": [...] },

    "assignee": { "accountId": "xxx" },

    "customfield_10015": "에픽 이름"  // Epic Name 필드

  }

}

```



**Task 생성**

```

POST /rest/api/3/issue

{

  "fields": {

    "project": { "key": "AEGIS" },

    "issuetype": { "name": "Task" },

    "summary": "제목",

    "description": { "type": "doc", "version": 1, "content": [...] },

    "assignee": { "accountId": "xxx" },

    "parent": { "key": "AEGIS-100" },  // 상위 Epic

    "customfield_10020": 123           // Sprint ID

  }

}

```

## 9. 복사-붙여넣기 처리



### 9.1 지원 형식



| 소스 | 구분자 | 처리 방식 |

|------|--------|-----------|

| Excel | Tab(\t) + 개행(\n) | 탭으로 컬럼 분리, 개행으로 행 분리 |

| Google Sheet | Tab(\t) + 개행(\n) | 동일 |

| Notion 표 | Tab(\t) + 개행(\n) | 동일 |

| Confluence 표 | Tab(\t) + 개행(\n) | 동일 |



### 9.2 처리 로직



```typescript

function handlePaste(event: ClipboardEvent, startRowIndex: number, startColIndex: number) {

  const text = event.clipboardData?.getData('text/plain');

  if (!text) return;

  

  const rows = text.split('\n').filter(row => row.trim());

  

  rows.forEach((row, rowOffset) => {

    const cells = row.split('\t');

    cells.forEach((cell, colOffset) => {

      const targetRow = startRowIndex + rowOffset;

      const targetCol = startColIndex + colOffset;

      // 해당 셀에 값 입력

      updateCell(targetRow, targetCol, cell.trim());

    });

  });

}

```
